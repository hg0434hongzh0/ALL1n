# ALL1n-通用POC渗透测试框架
为安全研究人员编写的漏洞本地验证库：一个基于Go语言和Fyne GUI框架开发的通用POC（Proof of Concept）渗透测试工具，支持树形结构管理POC、单点/批量测试、数据持久化等功能。

## 功能特性

### 🎯 核心功能
- **树形POC管理**：支持文件夹和POC的层级管理
- **单点测试**：针对单个POC进行精确测试
- **批量测试**：对选定范围内的所有POC进行批量测试
- **上下文感知**：智能识别测试范围，提高测试效率
- **POC传递方便**: 只需要复制poc_data.json即可完整copy队友的POC(doge
- **省去编写指定脚本时间**: 去你的pocsuite 去你的nuclei 去你的Yak(doge*2

### 📝 POC编辑器
- **完整字段支持**：方法、路径、参数、请求体、Headers、匹配规则等
- **多种Body类型**：支持Raw、JSON、Form格式
- **实时保存**：编辑后自动保存到本地

### 🗂️ 数据管理
- **自动持久化**：程序关闭时自动保存数据
- **数据导入/导出**：支持JSON格式的数据导入导出

### 🌐 网络请求
- **HTTP方法支持**：GET、POST、PUT、DELETE
- **超时控制**：内置请求超时和响应大小限制
- **自定义Headers**：支持添加自定义请求头
- **智能匹配**：根据匹配规则自动判断漏洞存在性

## 运行环境要求

### 系统要求
- **操作系统**：Windows 10+ / macOS 10.13+ / Linux
- **内存**：4GB RAM（推荐8GB）
- **存储空间**：100MB可用空间

### 软件依赖
- **Go语言**：1.21或更高版本
- **Fyne GUI库**：2.7.1
- **其他依赖**：程序会自动下载

### 基本操作

#### 1. 添加POC
1. 在左侧树形列表中选择一个文件夹
2. 点击"新建POC"按钮
3. 在右侧编辑器中填写POC信息
4. 点击"保存"按钮

#### 2. 组织POC
1. 点击"新建文件夹"创建系统分类
2. 通过拖拽（或删除/重建）重新组织POC结构
3. 文件夹支持多级嵌套

#### 3. 执行测试
1. 在顶部输入目标URL（如 `http://192.168.1.100:8080`）
2. 选择测试范围：
   - **单点测试**：选择单个POC节点后点击"单点测试"
   - **批量测试**：选择文件夹后点击"一键测试"
3. 查看底部日志区域的测试结果

#### 4. 数据管理
- **自动保存**：所有操作都会自动保存到 `poc_data.json`
- **手动导出**：点击"导出"按钮备份数据//有点多余
- **导入数据**：点击"导入"按钮恢复数据//有点多余

### POC字段说明

| 字段名 | 说明 | 示例 |
|--------|------|------|
| 名称 | POC显示名称 | KeyWord-SQL注入 |
| 方法 | HTTP请求方法 | GET、POST、PUT、DELETE |
| 路径 | 请求的相对路径 | `/api/login` |
| 参数 | URL查询参数 | `id=1&debug=true` |
| Body类型 | 请求体格式 | Raw、JSON、Form |
| Body | 请求体内容 | `{"user":"admin"}` |
| Headers | 自定义请求头 | `Cookie: session=abc123` |
| 匹配规则 | 响应中包含的字符串 | `Welcome Admin` |

## 注意事项

### 安全警告
⚠️ **仅用于安全研究**
- 本工具仅用于本地安全研究漏洞概念化验证
- 禁止用于非法入侵或攻击活动

### 技术限制
1. **网络请求**：默认超时时间为8秒，最大响应大小为1MB
2. **数据存储**：所有数据存储在本地 `poc_data.json` 文件中
3. **并发限制**：批量测试时每个请求间隔200ms，避免触发防护

### 故障排除

#### 常见问题
1. **程序启动失败**
   - 检查是否安装了正确的Go版本（1.21+）
   - 确保有足够的系统权限

2. **新建文件夹闪退**
   - 删除 `poc_data.json` 文件后重新启动//一般不会，最好不要
   - 确保程序对当前目录有写入权限

3. **网络请求失败**
   - 检查目标URL格式是否正确
   - 确认网络连接正常
   - 检查防火墙设置

#### 日志说明
- **INFO**：一般信息，如测试开始、进度等
- **VULN**：发现漏洞（响应匹配成功）
- **SAFE**：未发现漏洞（响应匹配失败）
- **ERR**：错误信息，如网络连接失败、请求超时等
